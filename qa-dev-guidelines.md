Dokumentasi Ujian Online menggabungkan QA checklist, struktur data JSON, dan catatan pengembangan. QA checklist mencakup registrasi dan login siswa serta admin, dashboard siswa dengan aturan dan tombol Mulai Ujian yang muncul saat exam-config.published = true, navigasi soal bebas dengan autosave di halaman ujian, submit menghasilkan skor dan penjelasan jawaban, serta halaman akun untuk ganti password atau hapus akun. Admin panel memungkinkan upload/edit soal, publish/unpublish ujian, monitoring leaderboard top 10, export data, dan reset leaderboard.

Struktur data JSON mencakup users.json (email, nama, passwordHash, role), admin-config.json (username, passwordHash), questions.json (examId, judul, daftar soal dengan id, tipe, teks, pilihan, jawaban benar, media opsional, penjelasan correct/incorrect), exam-config.json (examId, published, passingScore), serta examResults di localStorage yang memuat userEmail, examId, skor, dan detail jawaban.

Catatan pengembangan menekankan struktur folder yang rapi: HTML halaman siswa/admin dan komponen reusable, CSS termasuk main, animation, exam, admin, JS modul spesifik, dan JSON untuk data. Semua halaman wajib memanggil main.css dan animation.css serta app.js dan ui.js sebelum modul spesifik. Animasi dikontrol dengan atribut data-animate, dan feedback visual menggunakan ui.showToast() atau ui.showModal(). Jawaban siswa disimpan di localStorage, admin-only dapat mengubah soal dan config, password disimpan sebagai SHA-256 hash, email siswa harus .com, dan leaderboard hanya menampilkan top 10 skor.

Untuk debugging dan testing, gunakan showLoading() saat fetch atau save data, periksa struktur JSON dengan console.log, dan pastikan semua event listener di modal atau komponen di-reset saat modal ditutup. Dokumentasi ini membantu QA dan developer memahami alur ujian online serta integrasi komponen UI dan animasi.
